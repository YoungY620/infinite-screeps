# 经验教训记录

> 记录失败和成功的经验，帮助未来决策

## 格式说明

每条经验包含：
- **日期**: 发生时间
- **类型**: 成功/失败/警告
- **事件**: 发生了什么
- **原因**: 为什么发生
- **教训**: 学到了什么
- **状态**: 有效/已过时

---

## 📅 2026-01-06

### ⚠️ 警告：初始策略过于保守

**类型**: 警告  
**状态**: 有效

**事件**: 
- 初始代码只配置 4 个 Creep (2 harvester + 1 builder + 1 upgrader)
- 能量源利用率只有 20%，大量能量浪费

**原因**:
- 过于担心 CPU 限制
- 没有考虑保护期的激进发展窗口

**教训**:
- 保护期内应该最大化发展，不用担心防御
- CPU 20/tick 经过优化可以支持 10+ Creep
- 缓存目标 + reusePath 可大幅降低 CPU 消耗

**改进措施**:
- ✅ 更新代码为激进发展模式
- ✅ Creep 目标增加到 10 个
- ✅ 添加 CPU 优化（缓存、reusePath）
